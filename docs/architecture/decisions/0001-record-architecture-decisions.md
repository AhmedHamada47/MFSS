# ADR 0001: Record Architecture Decisions

## Status

Accepted

## Context

We need a consistent way to document architectural decisions for the MFSS project.

## Decision

We will use Architecture Decision Records (ADRs) as described by Michael Nygard.

## Consequences

- Architectural decisions will be documented in `docs/architecture/decisions/`
- Each ADR is a short markdown file with a unique number
- This makes onboarding easier and preserves context for future maintainers
